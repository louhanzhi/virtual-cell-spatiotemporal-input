## Talk 1 · 高章音（AI 虚拟细胞模型进展与评估分析，约 13:13 起）

非常感谢魏老师的介绍。那接下来就开始我的这个 talk，是一个搞 AI 的人的介绍，主要说一下我们最近在虚拟细胞方面的一些模型进展与评估的分析。

先讲一下这个大致的背景，就是我们这边 AI 侧定义的这个虚拟细胞到底是什么。从哪些角度去做的，其实前面很多生物学家都讨论过，生物学和 AI 科学家现在最大的一个共识，就是搭建一个在任何环境、任何刺激下，能够预测细胞未来状态的一个模型。这相当于是两个领域最大的共识。

所以我们在 AI 这一侧，也就是基于许多的公开数据来做这样一个细胞扰动的模型。然后大家会从很多的角度去搭建这个模型。比如说搞科学的会根据一些反应网络、动力学的参数来搭建在细胞模拟的模型，前几天在正刊上也有一篇工作就是这样去做的。然后还有一些科学家嘛，会利用这个细胞的结构以及细胞内部这些分子的空间状态、位置来搭建细胞模拟的模型。还有就是数据驱动的，我们 AI 侧主要做的其实是数据驱动的一个模型。

然后这个领域的进展大概分为几个阶段。一个是在 12 年左右，有一些在细胞建模的工作出来，那个时候大家在用一些规则驱动或者 ODE 的系统来建模细胞。然后 16~23 年的时候，随着 proteomics 这些流出技术的进步，提供了一个很好的数据底座，这个时间段测量技术在这里得到提升。然后在 22~25 年，就有很多细胞的 foundation model 开始出来了，比如说 scGPT、Geneformer、scFoundation [?] 这些细胞基础模型，它们利用前面积累的这些细胞数据，学了生物学的表征模型。

从大概 25 年年初开始，虚拟细胞这个词就一直很火，特别是 Arc 举办了第一届的 VCC challenge 之后，彻底在国际上掀起了这个话题。这个时候大家开始想想怎么把这些零散的工作平台化、可 benchmark 化。然后再前一些天，BioHub 也投了 5 亿美金，说是要去搭建一个虚拟生物学，可能从虚拟细胞上面再往上抽象了一层。

然后在细胞扰动这个层面，又我们同时已经有很多工作在如火展开了。比如说 Arc 他们做的 State 以及 STAC [?] 这些模型，然后 Mila、CellFlow，包括今天的 PerturbBench、PerturbFlow 这些模型。当前的话呢，达摩院之前发了电桥 [?]，然后清华大学和百图之前还发的细胞的 diffusion [?] 扰动模型，并且在 VCC challenge 上，百图他们拿了世界第一的成绩，等等等等。反正从去年到今年，这个词都是非常火的。

但是目前我们从 AI 侧也观测到了一些局限。比如说数据，现在大家主要还是从 scRNA 去做的。scRNA 的这个组学数据其实是很多的，但是它还是要缺一些其他的一些模拟，就是如说魏老师这边非常关心的蛋白质模拟，以及它们在空间和时间上的信息也是缺失的。然后就是跨细胞类型、跨疾病状态，加上 dose 信息，特别是在不同时间上，以及还有组合扰动，这些 OOD 的泛化能力，现在的模型都是非常弱的，没有模型能够解决这些跨域泛化的难题。

然后我们很多的模型呢，只是去强调"刷点"，把它的预测 performance 提高。但是它是不是真的理解了细胞里面的某些作用机制，比如说究竟是激活了哪一个开关分子、打开了什么通路，这里在 AI 的研究里面也是有很大空白的，这个可能生物学家们擅长，AI 的人还没有很好地去真的去做出一些 biological insightful 的东西。

然后在 benchmark 上面，虽然 Arc 他们举办了第一届比赛，但是他们的 benchmark 其实是有缺陷的。他们的一个指标是很容易受到攻击的——大家通过改变这些数据的 scale，其实它的 metric 就能够波动得非常大。所以怎么样跨越不同的工作去对比这些模型、真正做一个头对头的比较，从现在的角度还是比较困难的。

那么接下来就讲我们在这个领域做了哪些事情。我本来是想给大家主要介绍一下我们的基础模型的，但是呢，我们今天因为我要介绍这个东西，让同学用 Codex 去查了我们之前的代码，我们发现我们之前的评估是有问题的，根据 State、Arc 他们官方的 cell-eval [?] 去做的评估其实是有 bug 的。所以这里我不会介绍太多，就说一下大概的一个 idea 以及目前流出来的结果。比较 tricky 的是，其实这个领域里面很多的文章，我都不是特别确信他们的评估是不是 head to head 能够公平比较的。

然后在这里我说一下我们这个模型解决的问题。输入是一个 cell by gene 的矩阵，大家可以看到这个 A 图，每一行其实就是一个细胞，每一列其实代表里面有一个 RNA 的类型，里面颜色就是代表这个 RNA 流出来的表达量计数。这是细胞的原始状态，然后通过扰动（可能是药物 drug 的扰动，也可能是基因的沙除），预测的是它在扰动之后的状态——就这个细胞从 RNA 表达的层面，它的状态会发生哪种变化。是解决这样一个问题。

然后我们这个模型是一个 flow matching 的模型，是一个 latent 的 flow matching 模型，主要包含了一个 encoder 和一个 decoder。这个 by level 的 set encoder 和 set decoder 是我们专门设计的一个模型，其实是想要去解决这些细胞里面基因位置发生的置换不变性问题，就是不要把两列交换一下，结果就发生了实在不一样的变化。

然后我们的输入，这里这个 control cell 就是扰动前的，perturbation cell 就是扰动后的，这些 cell state 都会通过一个 encoder 编码到隐空间。然后在隐空间里，我们做了一个 latent 的 flow matching，把它从扰动前的状态映射到扰动后的状态。然后拿到扰动后状态的 embedding 之后呢，再通过一个 set decoder 就可以返回到它扰动后的真的 scRNA 表达矩阵。

然后这里面我们建模其实是在细胞的集合层面建模，这个可能和后面欣蓉 [?] 要介绍的 PerturbDiff 撇开沟工艺呢——就是我们建模的其实是一个细胞集合在扰动后发生的变化，而不是单个细胞。因为单个细胞它准确的变化是不靠谱的，因为这个细胞只要测量之后它就死掉了，所以它当天的没法找对到被观测到。

然后在这里面我们还专门基于 BioNLP [?] 这个分布式框架优化了一下训练过程，使得我们的实现会比 State 官方的实现至少要加速 10 倍以上。然后评估的协议我们是尽可能和 State 严格统一的，但是尽管严格统一，其实去做的时候还是会有很多问题。

然后这是我们最新流出来的结果，其实我们测了三个数据集——PBMC、Tahoe 100 million [?] 和 Replogle [?]，都是大家常用的一些数据。但是 PBMC 在这里没有放，因为 PBMC 其实是后面想要欣蓉 [?] 的，我们是从 PBMC 官网下载的数据，也处理了，然后通过 State 官方的 checkpoint 流出来的成绩和他论文里面差得太远了，而 State 官方没有给 PBMC 处理的代码以及 dataset，但是 Tahoe 100 million [?] 和 Replogle [?] State 官方是给了的，所以在这里我们是可以和他们做一个严格的头对头对标的。

然后做完这个对标之后，其实我们会发现，星号有标记的是我们严格跑赢。严格做这对齐的（官方），我们现在的状态和他们在一个数据集上是伯仲之间，在另外一个数据集上几乎所有情况都要更好。这里算法的部分，左边我就不具体介绍了。

在做这个基础模型的过程中，我们还有很重要的一条线，就是去 benchmark 这个细胞扰动，看看去测一下现在的模型到底到了什么阶段，以及看一下这个领域的发展状况究竟是什么样子。然后我们就做了一个 in the wild 的泛化能力测试。我们和之前的工作很不一样的一点是，我们的划分不是随机划分或者瞎划分的那种划分，而是专门用一个预训练的 STATE [?]（STATE 也是 Arc 他们官方前面出来的一个模型）的 embedding，在 embedding 级别根据相似度做了一个严格的划分，就是要看看这些模型在更难的情况下表现究竟会怎么样。

然后我们这里有三类场景，有 unseen cell、unseen perturbation，还做了一个非常独特的 unseen 场景——多个数据集的合并。因为我们发现每一个数据集，现在大家几乎都是在这个数据集上训一个模型，它很难做到通用。但其实我们的生物学家需要的是一个通用的模型，不管是任何场景、任何扰动，它都能有一定的预测能力。那这个时候我们最终的目标肯定是要把多个数据集融合在一起训练的，所以我们就做了融合训练的尝试，看看能不能提升模型效果。最终能不能让我们这个虚拟细胞扰动模型从不用泛化到通用，所以做了这样一个全新的 setting。

另外就是在模型接口这里，我们系统性地整合了 11 个模型。当时还没有 web coding 之类的，是我们同学人工把这 11 个模型的官方代码 reproduce、规范化，然后放在同一个框架上去做比较，也花了比较长的时间。然后就是一个多层的指标体系。我们发现，从 VCC 的比赛就可以看出来，它很多指标是相互冲突的——虽然这些指标都很重要，但他们拿的时候就是 A 好了 B 就不好、B 好了 C 就不好。我们把这个指标体系也分类梳理了一下，并且研究了一下不同指标之间的 correlation，最后推荐了大家最值得用的几个指标。

然后这里是 VCBench [?] 的一个关键发现。首先，我们是用 scFoundation [?] 的 embedding 做的划分。我们能够发现，用 embedding 做了严格划分之后，所有的模型相对于之前大家做的那种随机划分而言，都会出现明显的性能下降。这其实是提醒我们，之前的 OOD 评估可能还是过于乐观了。下面这张图，without scFoundation [?] 就是灰色的柱子（随机划分大家的表现），绿色的柱子是做了严格划分之后大家的表现，可以看到大部分模型效果都是下降的。反正第一部分的结果就是，我们发现随机划分会显著地高估此前 OOD 的泛化结果。

然后第二个结果，我们发现这些指标之间确实有一些明显的不一致甚至冲突——平均指标好的时候，它扰动特异性的指标反而不一定好。第三个表部分说的是平均指标，包含了几个实验，一个是单基因扰动的 single 实验，一个是预测在新药物扰动的 unseen 实验，还有一个是组合扰动（但组合里一个是已知、一个是未知），最后一个是两个组合扰动都未知。我们首先做了这五组实验，然后做出了 perturb 的 embedding（扰动向量的 embedding），给它们做了一个 similarity 矩阵。然后我们发现，其实只通过这个 perturb embedding similarity 矩阵就可以解释任务的难度，这个 distance 颜色越浅就说明 distance 越小、similarity 越高，那其实它的泛化能力就会越好，在这个表的 similarity 矩阵里面是能够映射上的。

下面这一部分是一个 DE 的扰动特异性指标。主要的发现就是这两个指标会比较不一致，有些模型比如 latent attitude [?] 在 PCC correlation [?] 上面，它们在两个指标上的排序都实在不同。这就是第二个结果，平均指标和扰动特异性指标其实是不一致的。

然后是第三个结果，我们做了一个跨数据集的实验。这里其实有两个 setting，一个是先把这些数据合并在一起，再取 HVG；还有一个是先取 HVG，然后再合并。主要的发现是，如果先取 HVG 的话，我们的 PCC 会相对更好，但是另外一个指标（隐空间层面的 SD 距离指标）会变得更差。然后在直接 merge 的时候，我们其实是发现模型效果是会变差的。然后我们就对这个数据做了一下分布的 scale——其实就是把两个数据集直接 merge，它们的数据分布肯定长得不一样，但我们通过一些 normalization 的方式来强行把它们的分布变得一致，看看会不会有提升。然后我们发现把它 scale 之后，对于 PD correlation [?] 和 SD 都是有帮助的。这其实又告诉我们，在数据合并的时候，是需要控制两个数据集之间的分布来进行对齐的。

然后我们做这个 benchmark 主要是想把这个领域的模型以及状态都摸一遍。对于虚拟细胞而言，可能我们这个 benchmark 本身就是一个基础设施。这里还有一些重要的点，我们发现线性模型在部分情况下会比复杂的模型更加稳定，是不是要用更好、参数更多的模型，其实取决于任务的规模和上下文信息，不同的模型在不同场景是高度场景依赖的，没有一个模型能稳定地在所有场景下都打败别人。在指标方面，我们也对不同指标进行了聚类，推荐了一些更符合生物学直觉的指标让大家使用，最后给了一个推荐矩阵。

然后这篇论文目前在送审到 Nature Machine Intelligence。在我们做这个 benchmark 的同时，我们也观察到这个领域有一些新工作出来，比如 Tahoe X1 [?] 和 xCell [?] 他们都发表出来了，用于更大规模的参数以及数据来训练虚拟细胞扰动模型。但是我们观测它们的结果时，其实感觉还是有点失望的。比如 Tahoe X1 [?] 从 70M [?] 扩大到 3B 的时候，它的 AUROC（你可以看第二张图）是从 0.632 扩大到了 0.638，只是小数点后第三位的一个 gap，其他几个数据上也是差不多的表现。同样这个 xCell [?] 也做了一个从几十 M 到 3B 的扩展，它想说它的 loss 是有一个 scaling law 的，但是我们如果不看 loss 直接看指标——看 DE 的斯皮尔森相关系数——可以看到它从小模型到大模型的过程中，指标提升也只是从 0.29 到 0.315，小数点后两位的变化。这其实说明这个领域目前还是非常难做的，或者说现在的数据基础设施是不足以让这些大模型 scale 上去的。这也是我们之前想要把多个数据集合并起来训练模型的一个 motivation。

总而言之，我对目前虚拟细胞领域这种纯 data driven、特别是基于 data specific（基于一两个数据集）去训练一个 foundation model 的范式，是不相信它的。就算它在当前数据集上能很好地 fit 当前数据集，但是换一个数据集它基本上马上就崩掉了，这是不能满足我们的生物学家需求的。所以我们判断未来这个虚拟细胞，它应该不是单个数据集的一个细胞，而应该打破不同数据之间的数据孤岛，使得不同的数据可以链接起来。比如说我们是不是可以把分子扰动、细胞状态、组织、微环境，甚至更多的数据有机地链接在一起，然后训练一个全尺度、全模态的模型，从而提高它的泛化性。

然后就是一个可信的 benchmark 体系。这个领域目前已经有了一些 benchmark，但即使是 VCC 比赛——在世界所认可的那个官方 benchmark——也是有问题的，首先它们的一个指标是相互冲突的，其次它们的每一个指标都是可以被人为攻击的，你可以专门去攻击这些指标从而取得最高的排名。然后就是可解释与实验闭环。目前我们做的都还是一个扰动的模型，但其实我们没办法解释这个细胞里面作用的机制，并且我们在 AI 侧做出来的模型，很多时候我们是找不到下游应用场景的——就只是刷了这么一个榜，但具体它解决了什么问题是不知道的。

所以我们后面工作的重心其实是专注在这一个点上去推进的，第二个点我们已经基本推进完了，然后下半年再围绕第一个重点推进。基本上就是这些。然后我讲得比较快。最后准备了一个问卷，各位专家对"AGI for 生命科学"如果有什么问题的话，可以扫码来填一下，然后我们后面就可以来进行一个聚焦圆桌的讨论。然后浦江科学社区的这些账号大家也可以关注一下。好，谢谢大家。

## Talk 2 · 袁欣蓉（PerturbDiff — Functional Diffusion for Single Cell Perturbation Modeling，约 35:00 起）

**魏天迪（介绍）：** 好，我现在分享一下我的屏幕。我们的下一位 speaker 是袁欣蓉 [?] 同学，她是蒙特利尔大学、Mila AI Institute 的博士生，师从彭健教授，本科是毕业于北京大学计算机科学学院，师从张铭 [?] 教授。研究方向主要围绕迁移学习与分布外泛化，关注大模型推理、智能体工作流、单细胞扰动建模、蛋白质结构表示学习以及知识图谱推理等问题，已经在 ICML、NeurIPS、ICLR 等国际会议发表多篇一作、共一的论文，其中包括 NeurIPS Spotlight、ICLR Oral 的工作。曾经在 Genentech、AveXis [?]、Regeneron 实验室、Amazon 和 Intel AI Labs 从事研究实习，曾获得 RBC Borealis [?] PhD Fellowship、蒙特利尔大学人工智能奖学金、北京大学优秀毕业生与优秀本科毕业论文等荣誉。好，欣蓉，要不你可以分享你的屏幕了。

**袁欣蓉：** 哦，好的好的，谢谢魏老师。非常感谢刚才魏老师的介绍。然后我可能就先开始讲今天想和大家分享的内容。我叫袁欣蓉 [?]。今天的这篇工作呢叫做 PerturbDiff，它完整的题目是 Functional Diffusion for Single Cell Perturbation Modeling，这个工作已经被 ICML 2026 接收为 poster 了。这篇工作的核心问题可以先用一句话概括：在单细胞的扰动建模里，我们不希望模型只预测一个平均的扰动后状态，而希望它能够生成一组合理的、带有细胞间异质性的反应分布。也就是说，当我们的输入是一个 control population、一个 perturbation 以及一些上下文的时候，我们输出的不应该是一个点估计，而应该有多个可能的 response。

那么接下来我就会先介绍 virtual cell 和药物发现的一些场景，然后解释为什么单细胞扰动问题天然是 distribution to distribution 的，最后进入 PerturbDiff 本身的 formal 方法，再最后介绍我们的实验结果以及一个简单的 takeaway。

我们先从 virtual cell 概念开始吧。这里我把 virtual cell 理解成 drug discovery process 里面非常重要的一个问题，因为它的一个直接应用场景就是做这个靶点发现。这一页展示的是从 disease understanding（疾病的理解）到 target validation（靶点发现），最后到药物的 design，一直到 lead 的优化，最后到我们的临床实验等等，以及到最后的上市阶段的一个完整流程。它可能每一步都非常慢，也非常贵。比如早期从靶点发现到验证的过程可能会非常慢，大概需要一年多，然后到后期的临床阶段，成本可能会到几亿到几十亿美金。现在大部分的制药公司，他们其实会优先选择一些已经被发现的靶点去解决一些已知的疾病。

那 virtual cell 在这个流程当中的价值呢，是让我们可以在真正的实验之前做一个大规模的 in silico screening。也就是说，我们在靶点发现这个阶段可以去模拟很多不同的 perturbation，然后去选择那些预测上最能够反映我们这个疾病发生的靶点。当然这并不是说 virtual cell 只能服务于早期的药物发现流程，当我们把靶点锁定之后，同样的建模思路其实也可以延伸到后续的分子设计，包括临床前的实验等等环节。

然后在这篇工作里面呢，我们其实关注的核心就是在靶点发现这样一个很重要的流程当中，我们抽象出一个最核心的研究问题，也就是给定一个 control 或者疾病的 cell state，我们知道有一些药物扰动或者基因扰动，我们想知道模型是否能够预测扰动之后的细胞群体分布。

那接下来简单介绍一下 virtual cell 整个的概念。现在基本上大家都会认为它是一个面向 virtual cell 模拟的 foundation model，或者说一个 world model。它接收的是三类输入，第一类是 control set，就是说我们健康的细胞它长什么样，或者是在某一个疾病状态下一些细胞群体长什么样；第二类是扰动的类型，比如细胞的某药物扰动，或者像 CRISPR 这种基因扰动等等；第三类是细胞的 context 状态，包括它的 cell type 是什么。它来源于哪些实验 batch。

那么在理想的情况下，virtual cell 它学到的不只是静态的基因表达表征，而是有更多潜在的生物和扰动的动态信息。也就是说，它需要知道我们的细胞当前处在什么状态，也要知道一个外部的干预会怎么样改变这个状态。然后我们的输出端也不只是一个平均的表达向量，我们真正想要的是一个扰动之后的细胞群体分布，因为这样我们能够尽可能地保留 cell 之间的多样性和随机性。因为在真实的实验当中，我们做一个扰动不会让所有的细胞都产生完全相同的反应，即使细胞非常相似。甚至接近一样，它也不太可能产生完全一样的反应。

然后有了这样的输出之后，我们可以进一步地从基因表达中读出我们想要的东西，比如它对应的功能，或者它对应的这些中间过程的动态信息，甚至包括基因的 pathway，以及到最后对应一个疾病的 phenotype。那么我们的 PerturbDiff 这篇工作其实就是一个更宽泛的 virtual cell 宏大愿景下的一个比较具体落地的工作。简单来说它是一个 conditional 的 generator，它的目标是生成可能的单细胞 perturbation distribution。

接下来我开始讲我们整个工作的核心 motivation。单细胞扰动建模其实有一个关键的事实基础，就是所有的细胞它都是没有配对的。在很多的监督学习问题里，我们其实希望会有一一对应的输入和输出，比如最简单的，同一个样本扰动前是什么样子、扰动后是什么样子。这样的事情在比如图像领域是很可行的，因为我们可以直接拿一个图片去做一些图片的编辑，然后得到一个新的图片，那这肯定是对同一个图片的变化。但是在 single-cell sequencing 这个领域里面呢，测序一般是破坏性的——一个细胞被测完之后，我们不能再对同样一个细胞施加另外的扰动并观测它之后的状态。所以左边的 control sample 和右边的 perturb sample，这两个细胞群体并不是同一批细胞的一一映射，我们观测到的是两个 cell 的 population，一个来自于 control 的 distribution，另外一个来自于 perturbed cell 的 distribution。

这就改变了我们想要建模的对象：我们不应该把任务理解成把每个 control cell 映射到某个对应的 perturbed cell，而更合理的对象是它们潜在的 cell distribution 长什么样。也就是说，在某一个 condition 下，我们单细胞 atlas 出现的概率分布到底是什么样。因此我们的监督信号其实是分布到分布的，而不是成对的 cell 到 cell 的。而我们模型主要想学的目标是，给定一个 control 的 distribution，以及 cell 的 context 和它对应的扰动类型，我们想要去预测最后的 perturb distribution 长什么样。

下一页说明了我们为什么要建模这样一个 cell distribution，然后这里面还有一个更深一层的 motivation 是说，即便是同样一个观测条件下的细胞群体，它可能对应的也不只是一个反应的 distribution。这页左边展示的是看起来相同的 context condition，比如说这些 cell 它们来自于 B 细胞这样的免疫细胞，但是它们底层可能也有不同的局部微环境状态。比如有一些细胞处于一个相对比较平缓的状态，有些细胞可能因为正在处于一个免疫状态当中。正在发生的是进行免疫的某些事，它可能处于一个比较 active 的状态。那这些状态一般来说在我们的实验数据里面是没有被完全观测到的，也不会有这样的数据 label，但它们会显著地影响细胞群体对某个扰动的反应。

而右边展示的是一个不同的从 control 到 perturbation 的变化。也就是说，即使我们用相同的 perturbation，而且拿同样的粗糙 cell context condition，我们最后得到的扰动 distribution 也可能不一样。比如这个地方，我用一个灰色的曲线表示这一个细胞群体，它们其实来自于相似的 cell distribution，但是因为一些微观条件、潜在因素不一样，所以它们存在也会基于这个灰色的 cell distribution 有一些偏离的变化。那在这样的情况下，我们面对的就不再是一个单一的 distribution，而是一些 distribution，也就是说我们需要去建模针对 distribution 本身的另外一个 distribution。

那模型其实需要承认并表达反应的不确定性和多样性。这个点其实是我们 PerturbDiff 用 diffusion 的一个动机之一，因为扩散模型比较擅长从条件输入中生成多个可能的样本，而在这个地方我们希望每一个样本不是单个的细胞，而是一个完整的 cell distribution。所以简单总结一下，这一页想表达，我们一些观测不到的细胞 context 信息其实也会显著地诱导分布上的多样性，如果我们的模型只能学到一个平均反应的话，就会把这种生物的差异性给抹掉了。

那接下来我会简单介绍一下我们整个 framework。基于第三个部分，我们想讨论的是，我们希望把这个 cell distribution 这个分布本身当成我们建模的随机变量，也就是说我们建模的对象不再是单细胞本身了，而是 cell distribution 这样一个函数。然后我们用的技术是一个叫做 kernel mean embedding 的数学工具，我们把这样一个函数分布映射到一个叫做 RKHS 的 kernel space 里面某一个点，然后在这个点的基础上搭建我们的扩散模型框架。

第二个部分，我们会讨论怎么样在这个 kernel space 里面，对这些映射过来的点（我们叫 embedding）搭建正确的扩散模型框架，以及对每个点（因为现在每个点是一个分布函数了）怎么样去对这个分布函数本身进行加噪和去噪。第三个部分是，我们会设计一个训练和采样的范式，让这个分布式的 objective 在有限的 batch 上的单细胞数据上可以进行代码实现，因为 kernel space 其实是一个无限维的 space，不太能够直接操作。第四个部分，我们会介绍我们的 model architecture。第五部分，我们会简单介绍一个很有意思的尝试，是用单细胞的 atlas 做一个预训练，把大规模的单细胞 atlas 作为一个 prior——直观上来说，如果模型已经见过很多各种的细胞状态了，那么在 perturbation data 比较少的情况下，它会更容易生成一些从生物角度上来说更能成为细胞的反应。

第一个部分，我们讲一下我们的建模对象。其实我们可以直观，如果想要把一个 cell distribution 这样一个函数当成随机变量的话，我们其实是在建模，给定我们的 control cell distribution 以及 cell 的 context 和对应的 perturbation 类型，让我们想要去预测最后这个 perturb cell 的 distribution 长什么样子。我们假设这样一个 distribution 可以表示为一个运动表征向量空间里面的高维概率分布。

那我们分为两步走，第一步是我们想去表示这样一个 cell distribution。我们表示的方法其实也很简单、很符合直觉，就是我们在现实当中观测到了一些 cell 的 batch，比如说我有 n 个细胞，它们组成了一个 batch，那么这 n 个细胞从数学的角度来说可以形成一个 empirical distribution，这个 empirical distribution 的定义其实就是一个以 Dirac [?] 形式去做的定义。第二步，在我们得到一个 cell distribution 的表示之后，我们看的是怎么样把这个 cell distribution 函数映射到一个可以直接进行扩散模型操作的对象上。我们采用的方法是把这个 cell space 里面的一个函数（如果是 empirical distribution 它就不是连续的，可能是一个离散的函数）映射到一个函数空间——一个 Hilbert space 里面。中间映射的方式就是我们会去做一个 kernel mean embedding 操作，把它映射到无限维的函数值空间里面。映射完了之后，我们可以想象每一个 cell distribution 函数就会被映射成这个数学空间里面的一个点，这个点和我们正常的欧几里得空间里面的一个向量其实是类似的。

然后我们这一节简单介绍一下我们整体的 PerturbDiff architecture。从高层来看，PerturbDiff 做的是一个 conditional 的 denoising，它的输入包含这几个部分，第一个是 perturbation 的信息，比如 perturbation 的 type，然后是 context 的信息，比如 cell type 和 batch 的 label，以及我们中间想做 diffusion 扰动的时候，我们会加一个加噪的噪声 time step。那么我们的模型会收到主要两个部分的 cell distribution 信息，第一个部分是 μ_c，μ_c 嘛是我们映射到 kernel space 之后的、表示 control cell distribution 的向量，然后 μ_t 嘛是我们扰动的 perturbation distribution 函数在 kernel space 的向量，而且是加噪之后的。那我们把这所有的信息都输入给我们的 diffusion 模型，然后经过一个 MMDiT 的框架，我们最后会输出得到我们预测的 μ_t 它对应的一个 μ_0 应该是什么样子。那我们这个 μ_0 其实就是我们真的 perturb cell distribution 的雏形。

然后在右边嘛，这个地方简单说一下，这个地方和普通的 cell level 的 diffusion 的区别在于，我们去噪的对象是一个 population 的 embedding，这个 embedding 是在 Hilbert space 里面的，它是一个无限维空间里面的对象，而不是一个单个细胞的表征向量。也就是说我们模型每一步它实在预测的是整个细胞群体应该长什么样子，而不是只预测一个细胞的表达向量。而右上角嘛，我们展示了 MMDiT framework 从 neural network 角度来说大致长什么样子，大致来说它分为两个 token stream（两条线）：一条线是 perturb 这条信息流，另外一条是 control 的信息流，它们经过 AdaLN 的 modularization 之后，会在 MMDiT 的内部进行一些信息融合和 attention，然后再分开；经历多次 MMDiT 不同 block 之间的 stack，把信息交融在不断。

这种结构对应的我们前面的问题设定，如果说我们的 hidden context 或者 control distribution 都会影响我们最后的 response，那我们的模型就必须得同时地观察到这些信息。那这一页简单总结一下，我们 PerturbDiff 其实是一个条件式的 denoising，它会把加噪之后的 cell population embedding 去噪到它真实的扰动 distribution embedding，中间会用到 control 的 distribution 信息、Diffusion time step，以及我们知道的 cell context 信息等等。

这一页数学比较多，但大家其实不用介绍的，它简单地把我们刚刚提到的 diffusion framework——它的状态、它想去去噪的状态长什么样子，中间怎么样加加噪（forward noisy），以及它最后要怎么样减去噪，得到我们最后的 diffusion 训练 loss——讲了一下。最后我们再去想怎么样把这个 loss 从 Hilbert space 里面的一个 norm（因为有一些数学性质的支持，我们能够把它翻译回我们原始（og）空间里面的一个 MMD loss）。相当于我们最后整个 PerturbDiff framework 用了两部分的 loss，第一个部分是 MMD term，这个 loss 主要是通过前面在 distribution level 的 diffusion 上推出来的，它在无穷地推导，这个部分主要是想把整个 cell population 当成一个随机向量去建模，第二个部分是我们加了一个 MSE term，这就是最经典的、和 DDPM 一样的 diffusion loss，这个 loss 在现实当中我们观测到它会在早期引导模型往我们想要的方向发展，是一个比较简单的 regularizer。

那我们最后其实是在三个不同类型的扰动数据上做了训练。第一种叫做 PBMC，是一个 cytokine [?] 的数据，它的规模大概是 10 个 million 的 cell，可能 90 多个 perturbation，然后有 18 个 cell type，这个数据测试的主要是免疫细胞在 cytokine [?] 这种蛋白刺激下的反应。第二个是 Tahoe 100 million [?]，对应的是一个药物扰动，它的规模特别大，可能有 100 多个 million，然后有 1000 多个 perturbation，覆盖 50 多个 cell line [?]，这个数据更接近我们药物扰动的应用场景。第三个是 Replogle [?]，对应的是基因扰动场景，也就是 CRISPRi，它有大概 0.6 million cell，相对其他两种类型的 cell 数据来说比较小，但它的 perturbation 类别很多，大概有 2000 多个，覆盖 4 个 cell line [?]。这三个数据集的生物机制和数据规模其实都不一样，分别代表不同类型的扰动，如果我们观测到同样一个模型在三个数据上都有效，那么说明这个模型并不是只针对某一个特定数据集调出来的。

然后我们经过实验发现，在三个数据集上，我们 PerturbDiff 基本上——我们去算了，这个雷达图里面表示了 12 个 metric，上届是 14 个 metric，这 12 个是只列出越高越好的 metric（因为这样更好绘雷达图），另外两个越低越好的 metric 放在论文附录里面了。那么我们可以从这个雷达图直观地看到，针对所有 metric 不同的方法是一个什么样的效果。这个地方绿色和蓝色的线是我们 PerturbDiff 的效果，对应了 fine-tune [?] 以及 pre-train 两种状态的表现。也就是说，从这种图的形状我们就大致可以看出，我们 PerturbDiff 覆盖了更大的 metric surface——换句话说，它不只是在某一个 metric 上特别好然后牺牲了其他指标，而是在大家现在 State 他们发现的所有这些 metric 上都取得了比较好的平衡。

然后尤其是那些对于显著表征基因的 metric，这些是我们真正去关心这些扰动是否恢复了那些应该被改变的基因（特别是一些叫做 differential expressed gene 显著被表达的基因）。那如果我们的模型只是平均表达上去接近的话，那么这些显著表达基因的变化方向和幅度是不对的，那么它们对我们后面的生物发现就会有损。但是我们发现我们的模型相对 State 而言，能够更有效地去发现这样和显著基因相关的 metric。这一页也进一步展示了我们的模型、State 以及真实数据它们在显著基因 recovery 上的分布。我们可以直观我们的 PerturbDiff 模型整个显著基因的 density 其实和真实数据更加接近，而相比较而言 State 就相差非常远了。无论是在显著基因上去看，还是在非显著基因上去看，都是这样子——我们的模型会显著地比 State 这个模型更容易、更好地去找到这样显著表达的一些基因。

第三，这一页我们主要想展示一下我们做的 ablation evidence。这页回答两个问题，第一个是我们的性能提升是不是来自于我们的 MMD loss，第二个是不是直接把模型变大（scale up）就可以了。左边是一个简单的 scaling evidence，我们可以直观这个 DE overlap [?] 这个 metric 包括 DE precision 这个 metric，它们随着模型从小到大、规模增大会进行一个提升，但是到了最大的模型的时候，它并不是总是继续变好，甚至可能会变差。也就是说，在 VC（virtual cell）场景上，单纯地扩大模型的 parameter 其实是不能赢的。右边的 ablation 展示了我们去掉 MMD term 之后，在 PBMC 和 Replogle 的雷达图上都出现了比较明显的下降。这和我们前面的建模逻辑是一致的，如果我们的目标是预测一个 cell distribution，那我们的 loss 里面必须要有直接对齐 distribution 的信号，换句话说就是 MMD term 是非常重要的，然后 MSE 它其实可以帮助模型去进行收敛。

然后这个地方也可以提一下，之前的 State 这篇工作其实也用了 MMD 这样一个 loss。相对来说，我们的 theoretical framework 可能是更严谨地揭示了为什么这样一个 MMD loss 是有用的，而且我们这个 framework 里面 MMD 的 loss 是只能用 MMD 的，但是在 State 他们的框架下，他们只是把 MMD 当成一个 population 之间的 matching 的方式，他们也可以用其他更多种的 population matching 方式去做。然后我们这篇工作在之前 rebuttal 的时候，也有 reviewer 提到，就是有没有可能去挖掘这个在 kernel space 里面建模的 framework 更多的潜能——因为我们现在可能还是在 cell space 里面去做一个类似 State MMD 的建模。然后我们当时也去进一步地把 kernel space 上一些特征拿去做了一个 perturbation variant [?] 的训练，把结果放在了文章附录里面。意思是，我们这个 kernel space 建模，在除了 MMD 这个 loss 的情况下，加上一些和 kernel space 相关的 loss，它也会更进一步地提升我们模型的效果。但这个地方因为时间有限，可能就没有展示这部分结果。

然后接下来一个很重要的点，是想和大家分享一下我们的预训练过程。这里的想法其实也比较简单、很直接，大规模的单细胞 atlas 里面，比如 sRNA [?]，它包含了大量各种的细胞类别，包括不同的细胞 batch（来自不同实验室的细胞 batch）。它们的状态，我们可以用这些大量的细胞来学到一个可能的细胞状态的 manifold。这个训练分两个部分，第一个部分是我们先在 single cell 上去学一个 general 的、或者叫 marginal 的 cell distribution，第二步是在这个 pre-train 的模型 checkpoint 基础上，再去在某一个扰动数据上做一个 fine-tune。

这个地方其实有一个表格，大家可以看到有一个明显的对比，single cell data 它会有大量的细胞，有 60 million，它的 cell type 会达到 600 多种，而相对来说扰动的数据一般只会涵盖如几个或者十几个 cell type，它的 diversity 程度远远是没有单细胞的细胞更大的。包括我们最后实验数据的 batches 信息，sRNA [?] 它会有 10K 多个不同的 cell batch，也就是说我们可以直观各种各样不同的、和 cell 相关的、和扰动的 noise，但是在扰动的数据里面，我们这个 batch size 会相对来说小很多。那我们想的是希望用这样一个 single cell 数据，去提升我们模型在见到更多、更 diverse 的 cell type 以及 batch 的情况下的效果，然后再用这种比较小的 perturbation data 去学从 control cell 到 perturbation 这样一个 shift 的过程。

然后我们后面会讲到，这个 marginal 的 distribution 是否真的能学到有用的 response。我们这页看到是一个 zero shot 的结果，也就是说我们在没有 perturbation 指导（没有 perturbation data）的情况下，我们 pretrain 的模型能不能比我们直接 random initializing 的模型表现更好。我们发现确实是这样子，它会明显好于 random initializing 的场景。换句话说，我们 pre-train 的时候，把模型从一个比较随机的状态推到了一个更好的 parameter state，这套 parameter state 可能能够让模型更好地把细胞对应的状态做一个对齐。

那这一页想展示的是，我们这个 pretrain 在很多场景上，比如 perturbation data 非常少的时候，会起到非常大的作用。比如右图里面，我们对 PBMC 这个数据做了 1%~5% 的 down sampling 去模拟一个数据匮乏的场景。我们会发现，如果直接从非常少的数据去做 from scratch 的训练，效果就非常差，但如果在 pretrain 的基础上去做一个 fine tuning，效果就会明显好特别多。而且这个效果也比较符合直觉，随着 data 的量取得越小，这个 gap 会变得更大，也就是说我们 fine-tune 它能够带来的提升会更大一些。

OK，那最后一页是一个简单的总结。我们这个 PerturbDiff 其实是想推动 virtual cell 的预测，从一个只聚焦于单个 cell distribution 的建模——无论是用一些 regression based 的模型（如 State 那样），或者用一些生成模型（如 CellFlow 这种 flow matching 的工作），或者做一些其他的工作，它们其实都是在把 cell 当成一个随机向量去建模，它们都默认我们最终想要建模的 cell distribution 只有一个，或者更准确地说，是在给定了 cell context 以及 perturbation type 这些条件下，对应的 cell distribution 只有一个。但是我们这篇文章想要去前面去讨论的一个问题是，perturbation prediction 它其实本质上是一个群体 level 的建模过程，而单细胞的实验里面也没有一一配对的细胞，所以我们其实应该更多去关注 distribution 到 distribution 的建模。而我们没有观测到的一些隐藏变量，很可能会诱导出分布级别的不确定性或多样性，即使在同样一个观测条件下——我们没有观测到的如微环境的改变。它的局部状态的改变，可能会导致非常不同的扰动反应。

那我们想要把这样的可能性直接放弃到放进我们模型的框架上。KME（kernel mean embedding）这个技术本身让我们这个 cell distribution 函数可以作为我们做扩散模型的状态，从而把有限的 cell sample 估计出来的 empirical distribution 分布嵌入到我们的 kernel space 里面，使得分布之间可以进行比较，最后做一个加噪和去噪的操作。那更严谨的数学证明过程，我们放在文章的正文（更多是放在 appendix 里面），大家有兴趣也可以一起去看一下的。然后最后一个点是 MMD 和 pre-training 这两个角度，都让这个方法变得更加可操作，因为我们的 MMD loss 会对齐整个 perturbation population，而我们用到的大规模 pre-training 能够提升泛化，而且能够显著地改善我们在一些用的数据场景上的表现。

所以我核心的一个 take away 其实就是，我们想要前面再宣传一种可能的想法，就是说我们未来可能建模的对象不再是 cell 本身，而是一个细胞的 distribution、是一个函数。那可能更遥远的未来，当我们真正要去构造所谓的 virtual cell 这个概念，甚至到最后的 virtual patient 的时候，我们从不同 level 去做建模，那可能不同 level 之间它都是一个有机联、有些动的关系。那可能我们最后建模的对象本身，就不再是单个的分子或者单个细胞的基因表征，而是很多异质性的、不同的对象组合在一起的一个集群本身，然后可能会对这样的 distribution 去进行一个总体的建模。对，这是我今天分享的我们这篇文章。
