# 论文骨架 · 细胞级时空 input

- **体裁**：perspective（观点综述），非 systematic review
- **字数**：1500–2000 字
- **thesis**：虚拟细胞的输入之争，本质是要不要承认细胞活在空间与时间里——而当前的数据范式恰恰把这两者都丢掉了。

> ⚠ 动笔前与老师确认：字数/格式是否照此、骨架可否调整、课程作业还是投稿、中文还是英文。

---

## 0. 引子（~150 字）
钩子：人人都说 virtual cell 要"多组学"作 input，但 scRNA-seq 要把组织解离成单细胞——这一步同时丢掉了空间（在哪、邻居是谁）和时间（从哪来、往哪去）。抛出 thesis。

## 1. 为什么时空数据是关键 input（~300 字）
- 论点：细胞功能是涌现的——由微环境（空间）和动力学（时间）共同决定，不在表达矩阵里。
- 弹药：Bunne「细胞跨时间和空间多尺度运行」；Nicheformer 证明只用解离数据训不出空间微环境复杂度。
- 收口：input 设计必须显式编码时空，而非寄望模型从静态快照里猜。

## 2. 空间轴：空间 input 该含什么（~400 字）
按"字段清单"写（区别于所有方法论文的地方）：
- 绝对坐标 (x, y[, z])
- 邻域图 / niche（谁在周围、组成）
- 细胞间通讯 CCI：配体-受体对（Spateo 的 CCI 框架）
- 形态 / 影像特征
- 技术落点：Nicheformer 怎么 tokenize 空间、CellPLM 位置编码喂坐标、Spateo 的 AnnData 规格（.obsm 坐标 / .X 表达 / .obs 注释）

## 3. 时间轴：时间 input 该含什么（~400 字）
抓住一个二分（来自 LivecellX 框架）：
- 伪时间 vs 真实时间：pseudotime ≠ 真实时钟
- 破坏性快照 vs 非破坏性活成像：测序类破坏性、只能反推（Waddington-OT、stVCR——尤其 stVCR 的坐标系对齐）；活成像（Caliban 追踪+谱系）非破坏、直接观测但通量低
- 字段：时间戳/伪时间、谱系 lineage、轨迹/速度、分裂-凋亡事件

## 4. 测量端的硬约束：三难困境（~300 字）
- 分辨率 × 通量 × 破坏性，三者不可兼得
- 弹药：测序类空间技术做到单细胞尺度就只剩几百-上千基因，全转录组+真单细胞分辨率今天拿不到；4D 活成像有时间但没分子深度
- 收口：理想时空 input 今天在物理上还不存在，数据议程必须正视这个边界

## 5. 结论 / 展望（~150 字）
给一份"理想时空 input 规格清单"（坐标+邻域+CCI+形态 | 真实时间+谱系+轨迹），指出缺口（跨平台坐标对齐、快照与活成像融合、时空标准化），呼应 Bunne 的社区议程，落回 thesis。

## 不可替代性（被问 novelty 时的答案）
没有任何一篇把这三件事合在一篇里讲：
1. 把空间 + 时间作为细胞级 input 的两条正交轴
2. 用"数据需求/规格"而非"模型能力"做锚
3. 接上测量端的分辨率-通量-破坏性三难（决定 input 能不能拿到）
中文圈更是空白，现有都是对 Bunne 那篇的科普转译。

## 一段开头草案（定语气，可改）
> 当我们谈论"虚拟细胞需要什么数据"时，答案几乎总是滑向同一份清单：转录组、蛋白组、表观组、代谢组——尺度越多越好，模态越全越好。但这份清单有一个被默认忽略的前提：它把每个细胞当成一管均质的分子溶液。真实的细胞不是这样存在的。它有一个地址（周围是谁、嵌在什么组织结构里），也有一段历史（从哪个状态来、正滑向哪个命运）。而我们最主流的输入数据——解离式单细胞测序——恰恰在制备的第一步，就把地址和历史一起冲掉了。本文想论证的是：对虚拟细胞而言，细胞层面的时空数据不是锦上添花的第五、第六种组学，而是决定模型上限的、被结构性低估的输入维度。
